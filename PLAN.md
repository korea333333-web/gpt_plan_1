# 달새김 (DalSaeGim) — 소중한 날을 달에 새기다

## 1. 프로젝트 비전

한국에는 음력 기념일 문화가 있다. 부모님 생신, 제사, 설날, 추석 — 모두 음력 기준이다.
하지만 요즘 종이 달력도 없고, 핸드폰 달력에 음력 표시도 부실하다.
매년 "올해 양력으로 며칠이지?" 찾아보다 잊어버리고 지나가는 일이 반복된다.

**달새김**은 이 문제를 해결한다.

기존 음력 앱들은 "달력을 펼쳐서 날짜를 찾아라"는 방식이다.
달새김은 다르다. **"내가 찾을 필요 없이, 알아서 알려준다."**

음력이든 양력이든 기념일을 등록하면, 매년 자동으로 양력 변환하고 미리 알림을 보내준다.

## 2. 핵심 기능 목록

### 2-1. 기념일 등록 및 관리
- 음력 / 양력 선택하여 기념일 등록
- 반복 주기 설정: 매년, 매달, 일회성
- 기념일 수정 / 삭제
- 기념일 카테고리: 생일, 제사, 기념일, 공휴일, 기타

### 2-2. 기본 내장 기념일
- 설날 (음력 1/1)
- 정월대보름 (음력 1/15)
- 석가탄신일 (음력 4/8)
- 단오 (음력 5/5)
- 추석 (음력 8/15)
- 사용자가 기본 기념일 on/off 가능

### 2-3. 메인 화면 — 다가오는 기념일 목록
- 달력이 아닌, 기념일 리스트가 메인
- 월별 그룹핑으로 다가오는 기념일 표시
- 음력 기념일은 자동으로 해당 연도 양력 날짜 변환하여 표시
- D-Day 카운트다운 표시 ("12일 남음")
- 몇 번째인지 자동 계산 ("칠순", "결혼 10주년")

### 2-4. 달력 보기 (보조 기능)
- 필요할 때만 열어서 확인
- 양력 달력에 음력 날짜 병기
- 등록된 기념일 표시
- 기본 화면이 아님 — 하단 탭에서 접근

### 2-5. 알림 시스템
- 기념일마다 최대 3회 알림
- 알림 시점 사용자 설정 (예: 7일 전, 3일 전, 당일)
- 기본값: 7일 전, 3일 전, 당일
- 알림 채널 3가지 (복수 선택 가능):
  - 카카오톡 (나에게 보내기)
  - 텔레그램 봇
  - 구글 캘린더 일정 등록

### 2-6. 기념일별 메모
- 각 기념일에 메모 작성 가능
- "작년에 뭐 했는지", "올해 뭐 준비할지" 기록
- 연도별 메모 히스토리 보관

### 2-7. 가족 공유
- 카카오톡 초대 링크로 가족 초대
- 가족 그룹 생성
- 그룹 내 기념일 공유 — 한 명이 등록하면 가족 모두 알림 수신
- 개인 기념일 / 공유 기념일 구분

### 2-8. 음력 ↔ 양력 간편 변환기
- 달력 열지 않고 바로 변환
- 음력 날짜 입력 → 양력 결과
- 양력 날짜 입력 → 음력 결과
- 특정 연도 지정 가능

## 4. 기술적 의사결정

### 4-1. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 14 (App Router) | React 기반, SSR 지원, Vercel 배포 최적화 |
| 스타일링 | Tailwind CSS | 빠른 UI 개발, 반응형 용이 |
| 백엔드/DB | Supabase | PostgreSQL 기반, 무료 티어, 인증/실시간 내장 |
| 인증 | Supabase Auth + 카카오/구글 OAuth | 소셜 로그인 간편 구현 |
| 음력 변환 | korean-lunar-calendar | npm 라이브러리, 한국 음력 정확도 높음 |
| 카카오 알림 | Kakao REST API (나에게 보내기) | 무료, 개인용 |
| 텔레그램 알림 | Telegram Bot API | 무료, 구현 간단 |
| 구글 캘린더 | Google Calendar API | OAuth 연동, 일정 자동 등록 |
| 스케줄러 | Vercel Cron Jobs / Supabase Edge Functions | 매일 알림 대상 체크 및 발송 |
| 버전 관리 | GitHub | 코드 관리, Vercel 자동 배포 트리거 |
| 배포 | Vercel | Next.js 최적 배포 환경, 무료 티어 |

### 4-2. 개발/배포 파이프라인

```
[로컬 개발] → [GitHub Push] → [Vercel 자동 빌드/배포]
                                      ↕
                               [Supabase DB/Auth]
```

1. **로컬 개발**: Next.js 개발 서버에서 코드 작성 및 테스트
2. **GitHub Push**: main 브랜치에 푸시하면 Vercel이 자동 감지
3. **Vercel 자동 배포**: 빌드 → 배포 자동 진행, 프리뷰 URL 생성
4. **Supabase 연동**: 환경 변수로 Supabase URL/Key 관리, DB 마이그레이션은 Supabase CLI

- GitHub 리포지토리: `dalsaegim` (프라이빗)
- Vercel 프로젝트: GitHub 리포와 연결, main 브랜치 자동 배포
- Supabase 프로젝트: Vercel 환경 변수에 연결 정보 등록

### 4-3. 라이선스/비용

| 항목 | 비용 |

## 5. 수익 모델

초기에는 무료 서비스로 운영.
향후 사용자가 늘어나면:
- 프리미엄 기능 (무제한 가족 그룹, 커스텀 알림 메시지 등)
- 광고 없는 버전

## 6. 대상 사용자

- 음력 기념일을 챙기고 싶지만 매번 날짜 확인이 번거로운 사람
- 부모님 생신, 제사일을 잊지 않고 싶은 자녀 세대
- 가족 기념일을 함께 관리하고 싶은 가정

## 7. 향후 확장 가능성

- 윤달 처리 고도화
- 기념일 추천 선물 연동
- 가족 앨범/일기 기능
- PWA로 앱스토어 등록 없이 모바일 앱처럼 사용
